# Tailwind CSS Rule Sections

This skill is divided by Tailwind docs-aligned categories.

## 1) Core (`core-`)

Start here for any Tailwind task:

- `core-utility-model.md` — utility-first principles, class composition, arbitrary values
- `core-theme-variables.md` — `@theme` directive, namespaces, override/extend, inline/static
- `core-responsive-and-states.md` — breakpoints, hover/focus/dark variants, `@custom-variant`
- `core-custom-styles.md` — `@layer`, `@utility`, component classes, base styles

## 2) Figma Workflow

Load when translating design tokens from Figma:

- `figma-to-theme-workflow.md` — step-by-step agent workflow for Figma CSS → Tailwind `@theme`

## 3) Performance (`perf-`)

Load for build configuration and optimization:

- `perf-purging-and-scanning.md` — content detection, source scanning, build output optimization

## 4) Accessibility (`a11y-`)

Load for accessibility and dark mode concerns:

- `a11y-and-dark-mode.md` — screen reader utilities, dark mode strategies, forced-color-adjust

## Suggested reading order

1. `core-utility-model.md` (foundation)
2. `core-theme-variables.md` (design tokens)
3. `core-responsive-and-states.md` (breakpoints, variants)
4. `core-custom-styles.md` (extending Tailwind)
5. `figma-to-theme-workflow.md` (when starting from a Figma design)
6. `perf-purging-and-scanning.md` + `a11y-and-dark-mode.md`
